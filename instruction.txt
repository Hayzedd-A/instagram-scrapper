I want to build an instagram scrapper using apify. 
The app is for a content creator to easily get to know all posted videos about a specific topic and to make some personal decision. 
The flow of the app is that, the frontend sends a query like: the search term, the recency, min views, min followers and then the app should return all neccessary data including the captions.
frontend sample request
{
  searchTerm: string,
  recencyDays: number,
  minViews: number,
  minFollowers: number,
  profile: {
    recencyPost: number,
  }
}


Step 1 ‚Äî Frontend creates job
POST /api/instagram/search


Backend:

Validate input
Create DB job record
Start Apify actor
Store runId internally

Return:

{
  "jobId": "internal_job_id",
  "status": "PROCESSING"
}

Step 2 ‚Äî Backend handles Apify fully

Backend:

Poll Apify run

When finished:
Fetch dataset
Filter
Normalize
Save result in DB
Update job status to SUCCESS


Step 3 ‚Äî Frontend polls backend
GET /api/instagram/job/:jobId


Response:

{
  "status": "PROCESSING"
}


or

{
  "status": "SUCCESS",
  "data": [...]
}



There is a new api to handle the scraping process, it is at /api/instagram/search
all payload should be converted to the format below 
{
  "searchTerm": "string",
  "limit": 100,
  "sortBy": "TOP_POSTS" | "RECENT_POSTS",
  "searchMode": "HASHTAG" | "USER",
  "recencyDays": number,
  "minViews": number,
  "minFollowers": number,
  "profile": {
    "recencyPost": number,
  }
}

this will return a result as 
{
  "jobId": "195979a0-acb6-4f68-817d-a3f7f3bdd3c2",
  "status": "PROCESSING"
}

then use the job id to check the status of the job at /api/instagram/job/:jobId
response could be 
{
  "status": "PROCESSING"
}
or
{
  "status": "FAILED",
  "error": "string"
}
