This is the new flow of the instagram route.

the search payload will be in this format
{
  "searchTerm": "string",
  "recencyDays": number,
  "minViews": number,
  "maxFollowers": number,
  "multiplier": number,
  "profile": {
    "recencyPost": number, (in number of weeks)
  }
}

multiplierFilter = videoPlayCount >= (average(videoPlayCount) * multiplier)

1. Use the searchTerm with the scrapeHashtag to get all related posts as much as possible. Maybe about 5000 or more.
2. filter them by the recencyDays, minViews, isCaptionEnglish, videoDuration > 55s, 
3. Save them to the database.
4. apply multiplierFilter.
5. fetch the profile data of all unique username with scrapeProfile from the filtered posts.
6. filter the profile data by the maxFollowers.
7. fetch profile reels with scrapeProfileReels from the filtered posts.
8. save the profile data to the database including their posts.
9. get getSearchJobStatus return structured grouped data of the profile post.


Note: 
1. Most apify service accepts array, process in batch to reduce time.
2. the scrapeProfileReels returns the posts of the profile with their transcript.
